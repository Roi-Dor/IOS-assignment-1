# CardBattle — Notes

## How to run
1. Open `CardBattle.xcodeproj` in Xcode 16 (or newer).
2. Select an iOS Simulator (e.g. iPhone 15, iOS 16+).
3. Press Run (⌘R). No code changes are required.
4. To test the East/West split: in the running Simulator, **Features → Location → Custom Location…**
   - Longitude `35.0` → East side
   - Longitude `34.0` → West side
5. To re-test the first-launch / permission flow, erase the simulator (**Device → Erase All Content and Settings**) or reset location privacy.

## Assumptions made
- **Project format:** Uses Xcode 16 file-system-synchronized groups (`objectVersion = 77`). Any `.swift` file placed under the `CardBattle/` folder is compiled automatically — no manual project edits needed when adding files.
- **Info.plist:** Auto-generated by Xcode (`GENERATE_INFOPLIST_FILE = YES`). The required `NSLocationWhenInUseUsageDescription` is injected via the `INFOPLIST_KEY_NSLocationWhenInUseUsageDescription` build setting, so the permission prompt appears on first run.
- **Bundle identifier:** `com.afeka.CardBattle` (change if you need a unique signing identity).
- **Min deployment target:** iOS 16.0. Swift 5.0 language mode.
- **Round timing:** Implemented with a cancellable `async` `Task`. Each round = 5s: cards face-up for 3s, face-down for 2s. The task is cancelled in `GameView.onDisappear`, so no scoring continues in the background.
- **Countdown indicator:** Shows seconds remaining in the round (5 → 0).
- **Scoreboard placement:** The human player's score sits on the left when assigned to the West side and on the right when assigned to the East side; PC takes the opposite side.
- **Tie rules:** A round with equal strengths awards no points but still counts toward the 10. A final tie resolves to PC (the house) — the Summary never shows a draw.
- **Name persistence:** Stored in `UserDefaults` under the key `playerName`. Empty on first launch (shows the "Insert name" button); the saved name is shown and remains editable on later launches.
- **Concurrency:** `GameState` and `LocationManager` are `@MainActor`; `CLLocationManagerDelegate` callbacks are `nonisolated` and hop back to the main actor to publish.

## Acceptance criteria mapping
- Location requested once and stopped after the first fix — `LocationManager.start()` calls `requestLocation()`; `didUpdateLocations` calls `stopUpdatingLocation()`.
- START hidden until name + side exist — `GameState.canStart`.
- 10 rounds then auto-navigate to Summary — `GameView.runGame()` → `endGame()`.
- BACK TO MENU resets scores/round — `GameState.resetToMenu()`.
